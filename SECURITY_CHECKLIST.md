# ChatALL 安全测试清单

## 一、CSP 策略测试

### 1.1 CSP 配置验证
- [ ] 确认 CSP 策略在生产环境正确配置
- [ ] 确认开发环境 CSP 策略允许 Vue Devtools
- [ ] 验证 `default-src` 只允许同源资源
- [ ] 验证 `script-src` 白名单仅包含必要的 CDN 和 Matomo
- [ ] 验证 `connect-src` 限制为已知的 Bot API 端点
- [ ] 验证 `frame-src` 设置为 `'none'`

### 1.2 CSP 违规监控
- [ ] 配置 CSP 违规报告端点
- [ ] 建立违规日志监控机制
- [ ] 设置违规告警通知

## 二、XSS 防护测试

### 2.1 输入验证测试
- [ ] 测试 HTML 标签注入（`<script>`, `<iframe>`）
- [ ] 测试 JavaScript 伪协议（`javascript:`）
- [ ] 测试事件处理器注入（`onclick`, `onload`）
- [ ] 测试 SVG XSS 攻击向量
- [ ] 测试 URL 注入攻击

### 2.2 输出编码测试
- [ ] 验证动态内容使用 `textContent` 而非 `innerHTML`
- [ ] 验证 Vue 模板自动转义功能正常
- [ ] 验证 Markdown 渲染库正确处理恶意输入
- [ ] 验证 API 响应内容正确转义

### 2.3 第三方脚本安全
- [ ] Geetest 脚本完整性校验
- [ ] 验证第三方脚本来源可信
- [ ] 定期审查第三方脚本变更

## 三、安全响应头测试

### 3.1 响应头配置验证
- [ ] `X-Frame-Options: DENY`
- [ ] `X-Content-Type-Options: nosniff`
- [ ] `X-XSS-Protection: 1; mode=block`
- [ ] `Referrer-Policy: strict-origin-when-cross-origin`
- [ ] `Permissions-Policy` 正确限制敏感 API

### 3.2 安全头覆盖范围
- [ ] 验证所有响应都包含安全头
- [ ] 验证安全头未被意外覆盖

## 四、Electron 安全配置测试

### 4.1 主进程安全
- [ ] 验证 `nodeIntegration` 设置（生产环境应禁用）
- [ ] 验证 `contextIsolation` 启用
- [ ] 验证 `webSecurity` 启用（除非有特殊需求）
- [ ] 验证 `sandbox` 启用

### 4.2 预加载脚本
- [ ] 验证预加载脚本最小化暴露 API
- [ ] 验证预加载脚本无敏感操作

### 4.3 新窗口安全
- [ ] 验证新窗口禁用 `nodeIntegration`
- [ ] 验证新窗口启用 `contextIsolation`
- [ ] 验证新窗口 URL 白名单机制

## 五、数据安全测试

### 5.1 敏感数据存储
- [ ] 验证 API Key 使用安全存储
- [ ] 验证敏感配置加密存储
- [ ] 验证日志中敏感数据被脱敏

### 5.2 网络传输安全
- [ ] 验证所有外部请求使用 HTTPS
- [ ] 验证 WebSocket 连接使用 wss://
- [ ] 验证证书有效性校验

### 5.3 Cookie 安全
- [ ] 验证敏感 Cookie 设置 `HttpOnly`
- [ ] 验证 Cookie 设置 `Secure` 属性
- [ ] 验证 Cookie `SameSite` 属性正确配置

## 六、依赖安全测试

### 6.1 依赖漏洞扫描
- [ ] 定期运行 `npm audit`
- [ ] 定期更新依赖版本
- [ ] 审查第三方库安全公告

### 6.2 依赖完整性
- [ ] 使用 `npm ci` 确保依赖版本一致性
- [ ] 配置 npm package-lock 完整性校验

## 七、渗透测试

- [ ] 使用 OWASP ZAP 进行漏洞扫描

- [ ] 使用 Snyk 进行依赖漏洞检测

### 7.2 手动测试
- [ ] 测试路径遍历攻击
- [ ] 测试本地文件读取
- [ ] 测试命令注入可能性
- [ ] 测试 SSRF 攻击向量

## 八、安全审计清单

### 8.1 代码审查要点
- [ ] 所有用户输入是否经过验证和转义

- [ ] 是否正确处理异常和错误信息
- [ ] 是否有适当的输入长度限制

### 8.2 合规检查
