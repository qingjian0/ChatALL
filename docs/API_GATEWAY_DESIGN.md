# ChatALL API 网关设计文档

## 1. 概述

本文档描述 ChatALL Web 应用的 API 网关架构设计，旨在提供安全、可靠、高性能的 API 请求路由和管理能力。

### 1.1 设计目标

| 目标 | 描述 |
|------|------|
| **安全性** | 提供完整的请求验证、速率限制和恶意请求检测 |
| **可扩展性** | 支持多后端服务、版本控制和负载均衡 |
| **可观测性** | 完善的日志、监控和错误追踪机制 |
| **高可用性** | 支持故障转移和备用代理 |
| **兼容性** | 兼容多种部署环境（Cloudflare、Vercel、Nginx） |

---

### 2.2 核心组件

| 组件 | 职责 | 实现方式 |
|------|------|----------|
| **请求路由器** | 路径匹配、版本控制、目标转发 | Cloudflare Workers / Nginx |
| **安全防护层** | 速率限制、请求验证、恶意检测 | 内置逻辑 + Cloudflare WAF |
| **日志监控** | 请求日志、错误追踪、性能监控 | Cloudflare Logs + Sentry |
| **负载均衡器** | 流量分发、健康检查、故障转移 | Cloudflare Load Balancing |

## 3. 请求路由设计

### 3.1 路径匹配规则

| 路径前缀 | 目标服务 | 说明 |
|----------|----------|------|
| `/api/v1/proxy/openai/*` | OpenAI API | OpenAI 接口代理 |
| `/api/v1/proxy/anthropic/*` | Anthropic API | Claude 接口代理 |
| `/api/v1/proxy/google/*` | Google API | Gemini 接口代理 |
| `/api/v1/proxy/groq/*` | Groq API | Groq 接口代理 |
| `/api/v1/proxy/cohere/*` | Cohere API | Cohere 接口代理 |
| `/api/v1/proxy/moonshot/*` | 月之暗面 API | Kimi 接口代理 |
| `/api/v1/proxy/baidu/*` | 百度 API | 文心一言接口代理 |
| `/api/v1/proxy/aliyun/*` | 阿里云 API | 通义千问接口代理 |
| `/api/v1/proxy/custom/*` | 用户自定义 | 自定义代理端点 |

### 3.2 版本控制策略

```
/api/v1/*    → 稳定版 (Stable)
/api/v2/*    → 开发版 (Beta)
/api/test/*  → 测试版 (Testing)
```

版本控制通过路径前缀实现，支持平滑升级和回滚。

### 3.3 路由优先级

1. **精确匹配**：完全匹配的路径优先
2. **前缀匹配**：按最长前缀匹配
3. **默认路由**：未匹配的请求转发到默认处理程序

## 4. 安全防护设计

### 4.1 速率限制策略

| 限制类型 | 配置值 | 说明 |
|----------|--------|------|
| **基础限制** | 100 请求/分钟/IP | 通用速率限制 |
| **严格限制** | 10 请求/分钟/IP | 敏感操作 |
| **突发限制** | 10 请求/秒/IP | 防止瞬时攻击 |
| **全局限制** | 10,000 请求/分钟 | 服务级限制 |

### 4.2 请求验证

#### 4.2.1 请求头验证

| 验证项 | 规则 | 错误码 |
|--------|------|--------|
| `Origin` | 必须在白名单中 | 403 Forbidden |
| `Content-Type` | 必须为 application/json | 415 Unsupported Media Type |
| `Content-Length` | 不超过 10MB | 413 Payload Too Large |
| `X-Request-Id` | 可选，但推荐 | - |

#### 4.2.2 请求体验证

- JSON 格式校验
- 字段类型和长度校验
- 敏感信息过滤（API Key 检测）

### 4.3 恶意请求检测

| 检测类型 | 检测方法 | 处理方式 |
|----------|----------|----------|
| **SQL 注入** | 正则匹配 SQL 关键字 | 拒绝请求 |
| **XSS 攻击** | 检测脚本标签 | 拒绝请求 |
| **路径遍历** | 检测 `../` 模式 | 拒绝请求 |
| **异常流量** | 机器学习模型分析 | 渐进式限流 |

## 5. 日志监控设计

### 5.1 请求日志格式

```json
{
  "requestId": "abc123def456",
  "timestamp": "2024-01-15T10:30:00Z",
  "method": "POST",
  "path": "/api/v1/proxy/openai/chat/completions",
  "origin": "https://chatall.app",
  "clientIp": "192.168.1.1",
  "statusCode": 200,
  "responseTime": 1250,
  "requestSize": 1024,
  "responseSize": 2048,
  "rateLimitRemaining": 95,
  "error": null
}
```

### 5.2 监控指标

| 指标 | 类型 | 用途 |
|------|------|------|
| `request_count` | 计数器 | 总请求数 |
| `error_count` | 计数器 | 错误请求数 |
| `response_time` | 直方图 | 响应时间分布 |
| `rate_limit_violations` | 计数器 | 速率限制违规次数 |
| `active_connections` | 仪表盘 | 当前活跃连接数 |

### 5.3 告警规则

| 告警条件 | 阈值 | 通知方式 |
|----------|------|----------|
| 错误率 > 5% | 持续 5 分钟 | Slack + 邮件 |
| 响应时间 > 5s | 持续 1 分钟 | Slack |
| 速率限制违规 > 100/min | 持续 1 分钟 | Slack |
| CPU 使用率 > 80% | 持续 5 分钟 | Slack + 邮件 |

## 7. 配置管理

### 7.1 环境变量

| 变量名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `ALLOWED_ORIGINS` | JSON | `[]` | 允许的源域名列表 |
| `ALLOWED_TARGETS` | JSON | `[]` | 允许的目标 API 列表 |
| `RATE_LIMIT_REQUESTS` | Number | 100 | 每分钟请求限制 |
| `MAX_REQUEST_SIZE` | Number | 10485760 | 最大请求大小 (10MB) |
| `REQUEST_TIMEOUT` | Number | 60 | 请求超时时间 (秒) |
| `LOG_LEVEL` | String | info | 日志级别 |

### 7.2 动态配置更新

支持通过环境变量或配置文件动态更新配置，无需重启服务。

## 8. 故障处理

### 8.1 故障转移策略

1. **自动重试**：临时故障自动重试 3 次
2. **备用代理**：主代理不可用时切换到备用代理
3. **熔断机制**：持续故障时自动熔断，定时恢复

### 8.2 降级策略

| 降级级别 | 触发条件 | 处理方式 |
|----------|----------|----------|
| **轻度降级** | 响应时间 > 2s | 限制并发数 |
| **中度降级** | 错误率 > 10% | 启用备用后端 |
| **重度降级** | 错误率 > 50% | 返回维护页面 |

## 9. 安全最佳实践

### 9.1 数据保护

- **敏感信息过滤**：自动检测并脱敏 API Key、Token
- **传输加密**：强制 HTTPS
- **请求签名**：支持可选的请求签名验证

### 9.2 访问控制

- **IP 白名单**：支持特定 IP 范围访问
- **Referer 验证**：验证请求来源
- **API 密钥管理**：支持 API 密钥验证
