# ChatALL CORS 代理与 API 网关 - 部署架构文档

## 1. 架构概述

本文档详细描述 ChatALL Web 应用的 CORS 代理与 API 网关部署架构，涵盖无服务器部署和传统部署两种模式。

#### 部署配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 部署平台 | Cloudflare Workers | 无服务器边缘计算平台 |
| 运行时 | Node.js 18 | JavaScript/TypeScript |
| 存储 | Cloudflare KV | 速率限制状态存储 |
| 网络 | Cloudflare CDN | 全球边缘网络 |
| 扩展性 | 自动扩缩容 | 根据请求量自动调整 |

#### 部署步骤

```bash
# 1. 安装 Wrangler CLI
npm install -g wrangler

# 2. 登录 Cloudflare
wrangler login

# 3. 配置项目
wrangler init chatall-cors-proxy

# 4. 部署到开发环境
wrangler dev

# 5. 部署到生产环境
wrangler deploy --env production
```

#### 服务器配置

| 角色 | 数量 | 规格 | 用途 |
|------|------|------|------|
| 负载均衡 | 2 | 2C4G | 流量分发、故障转移 |
| API 网关 | 3+ | 4C8G | 请求处理、代理转发 |
| Redis | 3 | 2C4G | 缓存、限流状态 |
| 监控 | 1 | 2C4G | 日志收集、监控告警 |

#### 部署步骤

```bash
# 1. 安装依赖
sudo apt update && sudo apt install nginx redis-server nodejs npm

# 2. 配置 Nginx
sudo cp deploy/nginx/chatall-proxy.conf /etc/nginx/conf.d/
sudo nginx -t && sudo systemctl restart nginx

# 3. 配置 Redis 集群
redis-cli CLUSTER INFO

# 4. 启动 Node.js 服务
cd api-gateway && npm start

# 5. 配置监控
prometheus --config.file=prometheus.yml
```

## 5. 安全架构

### 5.1 安全分层

```
┌─────────────────────────────────────────────────────────────────┐
│                      安全防护层                                │
│  ┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐  │
│  │   网络层安全    │ │   应用层安全    │ │   数据层安全    │  │
│  │ - WAF 防火墙   │ │ - CORS 验证     │ │ - 数据加密     │  │
│  │ - DDoS 防护    │ │ - 请求签名      │ │ - 敏感信息脱敏  │  │
│  │ - IP 白名单    │ │ - 输入验证      │ │ - 日志审计     │  │
│  └─────────────────┘ └─────────────────┘ └─────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 SSL/TLS 配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| TLS 版本 | TLS 1.2, TLS 1.3 | 禁用 TLS 1.0/1.1 |
| 证书类型 | ECDSA 证书 | 更安全、性能更好 |
| HSTS | max-age=31536000 | 强制 HTTPS |
| OCSP Stapling | 启用 | 证书状态验证 |

## 7. 高可用性设计

### 7.1 故障转移策略

| 故障类型 | 检测方式 | 恢复策略 | RTO |
|----------|----------|----------|-----|
| 节点故障 | 健康检查 | 自动剔除故障节点 | < 30s |

| 服务故障 | 熔断机制 | 降级到备用服务 | < 1min |
| 数据中心故障 | 多区域部署 | 切换到备用区域 | < 5min |
